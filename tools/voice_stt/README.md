# Sokol voice STT prototype

Рабочий Windows-прототип: лёгкий Silero VAD локально выделяет и обрезает фразу,
`gpt-4o-mini-transcribe` распознаёт текст, WeSpeaker локально определяет голос,
Supertonic произносит только короткое подтверждение. STT не исполняет команды:
он передаёт распознанный текст будущему мозгу.

Whisper исключён из обычного тракта. Он загружается только при недоступном API
и распознаёт лишь аварийные команды `стоп`, `домой` и `следуй за мной`.

Пока робот говорит через этот процесс, захваченные фрагменты блокируются и не
отправляются в API. Это защищает от распознавания собственного голоса, но до
подключения AEC человек, говорящий одновременно с роботом, тоже будет заглушён.
Музыку также нужно проигрывать через общий аудиотракт, чтобы он видел её эталон.

## Первый запуск

1. Запусти `install_voice_stt.bat`.
2. Создай рядом файл `audio_secrets.txt` с ключом проекта OpenAI.
3. Запусти `start_voice_stt.bat`. Калибровка тишины больше не нужна.

Ключи, профили, WAV, кэши, транскрипты и статистика исключены из Git.
Запись нового голосового профиля должна запускаться мозгом через структурированный
вызов `PhraseWorker.request_voice_enrollment(name)`, а не разбором текста STT.

Это пока настольный прототип. Для Home Assistant потребуется передать контейнеру
аудиоустройство и заменить Windows BAT-файлы командой запуска add-on.
