# Рабочие контуры робота

Этот файл — короткая операционная карта для следующей сессии. Не заменять
рабочие контуры тестовыми GPIO-сканами, заглушками камеры или прошивкой STM32,
вшитой в образ S3.

## Прошивки

| Что меняется | Где собирать | Как обновлять | Что не делать |
| --- | --- | --- | --- |
| STM32F071 | `D:\w_space\V7s_Plus` | `build_stm32.bat`, затем временная загрузка BIN на уже установленный S3 и HA-кнопка | Не запускать `compile.bat` и OTA S3 ради STM32; не заводить слоты/встроенные BIN | 
| ESP32-S3 `v7s-plus` | этот репозиторий | `compile.bat`, затем `flash.bat <host>` | Не путать с STM32-обновлением |
| ESP32 audio bridge | его отдельный репозиторий | его собственный OTA/serial workflow | Не подменять его прошивкой S3 |

Подробности STM32: [STM32_UPDATE.md](STM32_UPDATE.md). При изменении STM32
`.ioc` обязателен CubeMX generate перед сборкой; ручная настройка PWM только в
коде недопустима, если таймер/пин должны быть назначены CubeMX.

## Зарядка и ИК

Реальный выход управления DC/DC — `PC7 / TIM3_CH2`, PWM 20 кГц, активный
высокий уровень. Он задаётся в `V7s_Plus.ioc` как `CHARGER_ENABLE_PWM`.
Автономная логика зарядки живёт на STM32: наличие дока, debounce, CC/CV и
защиты не зависят от Home Assistant или доступности S3. HA оставляет ручные
контролы для отладки заезда и телеметрию, но не является обязательной частью
безопасной зарядки.

Ток батареи Modbus — знаковый milliampere: отрицательное значение означает
ток заряда, положительное — разряд. Не трактовать `uint16` как амперы и не
подменять его тестовой константой. ИК-приёмники должны публиковать
отфильтрованные уровни/состояния, а не сырые короткие окна как конечный сигнал
стыковки.

## GoPro: видео, управление и звук

```
GoPro HERO12 ── RTMP напрямую ──> HA go2rtc (gopro_robot) ──> camera.GoPro Robot / dashboard
       │
       └── BLE-команды через временный Windows bridge ──> включение, сон, start/stop, Wi‑Fi
```

Камера поддерживает 5 ГГц. Домашняя сеть `Fermiums` — 5 ГГц; `Fermiums_2.4` —
2,4 ГГц. SSID и пароль не хранятся в Git. Для передачи команд GoPro должна
быть включена и находиться на экране **Connections → Pair/Connect**: вне него
HERO12 закрывает BLE service управления, поэтому честная команда обязана
вернуть ошибку, а не имитировать успех.

Постоянный исходник временного моста: `tools/gopro_ble_bridge.py`. Он получает
адрес камеры, RTMP URL, токен и данные домашней сети из окружения Windows. В
Home Assistant кнопки вызывают этот bridge по LAN; карточка камеры использует
реальный поток `gopro_robot`, а не заглушку. После перевода на 5 ГГц RTMP
кратко оборвётся и должен подняться в том же `gopro_robot`.

Аудио GoPro идёт вместе с RTMP. Для «звук → текст» следующий рабочий контур:

```
RTMP audio -> ffmpeg/audio consumer -> Whisper (HA add-on) -> текстовая сущность или событие HA
```

Не заявлять распознавание готовым, пока не проверены наличие аудиодорожки у
живого RTMP и появление фактического текста от Whisper.

## Перед коммитом

1. `git status --short` — без паролей, токенов, BIN, дампов и `.codex_*`.
2. `git diff --check`.
3. Собрать только затронутую прошивку.
4. Закоммитить исходники и документацию; не коммитить generated output.
