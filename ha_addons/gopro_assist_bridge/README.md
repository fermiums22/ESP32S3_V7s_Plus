# GoPro Assist Bridge

Локальный Home Assistant add-on получает AAC-аудио из RTSP-потока GoPro,
декодирует его через ffmpeg в PCM 16 kHz mono, режет речь адаптивным RMS VAD и
отправляет сегменты в штатный Assist WebSocket API.

Команда принимается только при наличии позывного `Сокол` или `Сокол девять`. После
распознавания фраза активации удаляется, а оставшийся текст передаётся в тот же
Assist pipeline на этап intent. Ответ произносится через `tts.speak` и аудиоплеер
робота.

По умолчанию управление проходит через выбранный Conversation agent. Опциональный
прямой режим использует OpenAI Responses API, умеет читать сущности, вызывать любые
сервисы Home Assistant и смотреть через `camera.robot_eyes`. Расходы ограничиваются
локальными дневным и месячным бюджетами; Whisper и TTS остаются локальными.

Команды `следуй за мной`, `стой` и `езжай домой` обрабатываются локально и не
тратят токены. Следование использует три откалиброванные дистанции STM32/S3,
ограничено 22 rpm и отключается при потере цели или любом защитном сигнале.
Профили голосов хранятся только в `/data/speaker_profiles.json`; это лёгкое
семейное различение говорящих, не биометрическая аутентификация.

Встроенный системный характер подключается через `agent_system_prompt: @builtin`.
Он различает роли Виктора, жены, дочки, гостей, Алисы и Мики, задаёт манеру юмора,
локальные «желания» робота и границы инициативы без разрешения опасных действий.
